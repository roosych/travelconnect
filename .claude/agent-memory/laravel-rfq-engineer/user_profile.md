Building a B2B inbound tourism platform as the product owner/lead developer. Wants a clean, operator-centric monolith — no enterprise patterns, no microservices.

Primary audience for the running system is the operator (a tour operator business), who manages the full RFQ lifecycle on behalf of agencies. Agencies and suppliers are secondary actors.

Values: correctness of domain rules over code elegance, minimal abstractions, Laravel idioms first. Spec is authoritative — follow it exactly, note any deviations with justification.
